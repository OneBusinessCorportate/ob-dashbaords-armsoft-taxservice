# OneBusiness — Дашборд расхождений (ArmSoft / TaxService / выгрузка Артёма)

Mobile-first веб-дашборд для отслеживания расхождений между:

- реестром клиентов OneBusiness (`ob_accounting_companies`),
- налоговым кабинетом **TaxService** (`v_tax_accounts`, данные Артёма),
- **ArmSoft** (`v_armsoft_companies`, данные Артёма),
- списком-выгрузкой **Артёма** (`artem_companies`),
- отчётами бухгалтеров с утренних встреч (`accountant_daily_comments`).

Backend — существующий Supabase-проект **OB Artyom** (`rbtvbsbcycdlwmrzjwun`).
Чистый HTML/CSS/JS (без фреймворков); Supabase JS v2 лежит локально в
`public/js/vendor/`. Vite используется только как dev-сервер и для сборки.

## Возможности

1. **Сводка** — карточки-показатели (дельта сегодня, нет в TaxService/ArmSoft,
   проблемы выгрузки, ждут проверки, исправлено со вчера, новые за сегодня)
   и таблица динамики по дням: видно, уменьшается ли дельта после каждой
   выгрузки Артёма (▼ лучше / ▲ хуже / = без изменений).
2. **Дельта** — каждый показатель кликабелен: открывается список конкретных
   компаний с ՀՎՀՀ, бухгалтером, признаками «где существует», причиной,
   датами и источником.
3. **Проверка с Эмилией** — для каждого расхождения: 7 статусов подтверждения,
   комментарий, ответственный, приоритет. Всё сохраняется в Supabase
   (`delta_items`), не теряется после обновления страницы.
4. **ТЗ Артёму** — автоматически собирает расхождения со статусом
   «Проблема выгрузки Артёма» и генерирует готовое русскоязычное сообщение
   для копирования.
5. **Встречи** — сравнение того, что бухгалтеры отчитали на утренних встречах,
   с выгрузкой Артёма, TaxService и ArmSoft (включая неточные совпадения «≈»).

## Запуск

```bash
npm install
PORT=8080 npm start      # dev-сервер (vite), → http://localhost:8080
npm run build            # production-сборка в dist/
```

Сайт полностью статический (supabase-js лежит локально в
`public/js/vendor/` — без зависимости от CDN), поэтому подойдёт и любой
статический хостинг: Render (`npm start` с `$PORT`), GitHub Pages, Netlify,
Vercel — или просто `python3 -m http.server` из корня. Ключ в
`public/js/config.js` — публичный anon-ключ Supabase (это нормально:
он предназначен для браузера).

## Бизнес-логика (настраивается в `public/js/config.js` → `RULES`)

В реестре клиентов **нет собственного поля ՀՎՀՀ** — оно восстанавливается
через связь `tax_account_id → v_tax_accounts.tin` или совпадение названия.

- **Должен быть в TaxService**: клиент активен **и** (известен ՀՎՀՀ **или**
  заполнен `tax_account_id`).
- **Должен быть в ArmSoft**: клиент активен **и** заполнено ArmSoft-поле
  реестра (`armsoft_company_id`) — лучший доступный признак «у клиента есть
  учётка ArmSoft». Правило — обычная JS-функция, легко изменить.

Типы расхождений (любой можно отключить в `RULES.ENABLED_ISSUE_TYPES`):

| Тип | Смысл |
|---|---|
| `missing_taxservice` | активен + ՀՎՀՀ, но нет в TaxService |
| `missing_armsoft` | ожидается в ArmSoft, но нет в ArmSoft |
| `tax_not_in_artem` | есть в TaxService, нет в выгрузке Артёма |
| `armsoft_not_in_artem` | есть в ArmSoft, нет в выгрузке Артёма |
| `meeting_not_in_artem` | упомянута бухгалтером, нет в выгрузке Артёма |
| `artem_without_work` | в выгрузке Артёма, но нет ни активности, ни упоминаний |

**Сопоставление**: сначала точное по ՀՎՀՀ → затем по нормализованному
названию (lowercase, без кавычек/пунктуации, без ООО/LLC/ՍՊԸ/ԱՁ/ИП, схлопнутые
пробелы) → затем нечёткое (транслит кириллица↔латиница). Нечёткие совпадения
помечаются «≈» и **не** считаются точными.

## Данные в Supabase (созданы этой задачей, миграция аддитивная)

- `daily_delta_snapshots` — один снимок в день (уникальна `snapshot_date`);
  `delta_change` = сравнение с предыдущим днём.
- `delta_items` — живой реестр расхождений. Уникальный `issue_key`
  (тип + ՀՎՀՀ/название) сохраняет статусы проверки при пересчётах;
  исчезнувшие расхождения закрываются `resolved_at`, вернувшиеся —
  переоткрываются.
- `artyom_tz_items` — задачи Артёму (автозаполнение при статусе
  «Проблема выгрузки Артёма»).
- `v_artyom_export_meta` — время последнего прогона парсеров Артёма
  и количества записей.

Пересчёт выполняется кнопкой «⟳ Пересчитать» (и автоматически один раз в день
при открытии). Существующие таблицы проекта не изменялись.

## Безопасность

Инструмент внутренний: как и остальные таблицы проекта, новые таблицы доступны
anon-ключу, но у новых таблиц RLS **включён** с явными политиками — при
необходимости доступ легко ужесточить (например, только `authenticated`).
⚠ У пяти старых таблиц проекта (`ob_accounting_companies`,
`accountant_daily_comments`, `artem_companies`, `accounting_activities`,
`document_records`) RLS выключен — это существующее состояние проекта,
данная задача его не меняла.
