# Журнал изменений

Запись формата «что / зачем / результат» обязательна для каждого PR, меняющего код: без неё робот `doc-sync` краснит и мердж заблокирован. Форточка для заведомо тривиальной правки — метка PR `skip-changelog`.

## 2026-07-24
- Фикс «Сохранить день» (Отчёт по дням): сохранение падало с `invalid input syntax for type integer: "46.8"`. Причина: колонка `accountant_day_reports.export_minutes` была `INTEGER`, а дневной итог минут дробный (норматив на услугу дробный — счёт 7,8 мин, поэтому напр. 6×7,8 = 46,8). Расширил тип колонки до `numeric` (миграция `sql/2026-07-24_day_report_export_minutes_numeric.sql`, применена к проекту OB Artyom; определение в `sql/2026-07-21_…` тоже поправлено на NUMERIC для новых установок). Зачем: обратная связь бухгалтера за день должна сохраняться. Результат: день с дробным итогом минут сохраняется (проверено значением 46,8 → HTTP 201). Покрыто детерминированным автотестом в `tests/run.js` (итог 6×7,8 = 46,8 дробный и переживает JSON-round-trip без потери точности).
- Поясняющие тултипы по всему дашборду: `data-tip` на каждом столбце, «большой цифре» (плитки/карточки), строке-метрике и заголовке-блоке во всех разделах (Сводка, Выгрузки, Синхронизация, Дельта, Проверка, ТЗ, Бухгалтеры, Отчёт по дням, Встречи, Созвоны) + отдельная страница `daily.html`. В подсказках — что это, источник (таблица/вью/RPC) и формула, а также как исключаются мусорные строки и обратные расхождения. Зачем: чтобы по каждой цифре было видно, откуда она и как считается, и можно было задавать целевые вопросы. Результат: движок подсказок добавлен и на `daily.html` (`initTooltips` + стили в `daily.css`); значок ⓘ показывает, где есть пояснение.
- Правило в `AGENTS.md`: тултипы обязательны и синхронны с кодом — при изменении смысла/формулы цифры её `data-tip` обновляется в том же PR (при каждом пуше). Зачем: пояснения не должны устаревать. Результат: проверяется в ревью.
- Селект бухгалтеров на странице «Утренние созвоны» теперь показывает ВСЕХ бухгалтеров реестра OB (объединение с теми, кто говорил на созвонах), а не только участников созвонов. Зачем: в фильтре не хватало части бухгалтеров. Результат: доступен каждый бухгалтер; при отсутствии созвонов показывается пустое состояние.
